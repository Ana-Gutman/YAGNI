# ADR003: Base de Datos Compartida
**Estado:** Aceptada  
**Participantes:** Federica Cabrera y Ana Gutman  

## Contexto
Una arquitectura centralizada reduce la complejidad inicial y asegura consistencia transaccional entre módulos clave del sistema, como UsuariosClientes, PedidosProductos e Inventario. Se busca simplicidad para ajustarse a los límites de tiempo para la entrega del prototipo.

## Decisión
Utilizar una única base de datos MySQL compartida por los módulos principales.

## Opciones Consideradas
- **Bases de datos independientes:** Rechazada por la mayor complejidad en sincronización y sobrecarga operativa.
- **Base de datos compartida:** Aceptada por simplicidad y control transaccional.

## Consecuencias
- **Beneficios:** Garantiza la consistencia de los datos entre módulos, simplifica el mantenimiento y reduce la complejidad del desarrollo.
- **Costos:** Limita la escalabilidad horizontal a largo plazo y podría ser un cuello de botella en el sistemas si maneja alta concurrencia.
